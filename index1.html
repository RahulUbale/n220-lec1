3. External JavaScript (Best Practice)

JavaScript is usually kept in separate .js files and linked with the <script> tag.

Example:

<script src="script.js"></script>

Why use external files? - Cleaner and more organized code. - Reusable across multiple pages. - Easier to maintain and update. - Improves page speed when cached.

4. Placement of Script Code

Choosing where to place your JavaScript code inside an HTML file is very important, because it directly affects how quickly the page loads and how your code behaves. There are several placement options, but not all are recommended:

  • Inside <head> (without attributes): - The browser will pause HTML parsing to load and run the script immediately. - This blocks the page content from rendering until the script finishes. - As a result, users may see a blank page for longer if the script is large or external. - Not recommended for most cases.
  • Inside <head> with defer: - Scripts are downloaded in parallel while the HTML continues to load. - Execution is delayed until after the HTML is fully parsed. - Maintains the original order of multiple scripts. - Best choice if you want your script logically in the head but executed after parsing.
  • Inside <head> with async: - Scripts are downloaded in parallel and executed as soon as they are ready, without waiting for HTML parsing to finish. - Good for independent scripts (like analytics or ads) that don’t depend on other scripts or the DOM. - Can cause issues if order matters, since scripts may run unpredictably.
  • At the end of <body> (before closing </body>): - The most common and recommended placement. - The browser loads and displays all HTML first, so the user sees the page quickly. - Then the JavaScript executes, ensuring that DOM elements already exist when scripts run. - Especially useful if your script manipulates or interacts with page elements.
  • Inline Event Attributes (like onclick): - Example: <button onclick="alert('Hello!')">Click</button>. - Convenient for very small, one-off tasks. - Not recommended for maintainable code, since it mixes HTML with JavaScript and is hard to scale.

Summary of Best Practice: Place scripts at the end of the <body> section for most use cases. Use defer in the <head> if you want better control and clean separation without blocking HTML rendering. Reserve async for independent scripts that don’t rely on order. Avoid plain head scripts and inline event attributes in professional projects.
