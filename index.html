JavaScript Fundamentals

A Hands-On Lesson for Absolute Beginners

What is JavaScript?

JavaScript is a programming language that makes websites interactive. Think of it as the "brain" of a website. It allows us to:

  • Perform calculations.
  • Respond to user actions (like a button click).
  • Update the content of a page without reloading it.

1. How to Comment in JavaScript

Comments are notes that programmers write to explain their code. The computer ignores them completely. There are two types of comments:

  • **Single-line comments:** Start with two forward slashes `//`.
  • **Multi-line comments:** Start with `/*` and end with `*/`.
// This is a single-line comment. It's great for short notes.

/*
This is a multi-line comment.
You can use it to write longer explanations
that span multiple lines.
*/

2. Variables: Storing Data

A variable is like a container that holds a value. We use the keyword `let` to create a variable and the `=` sign to give it a value.

// We declare a variable named 'studentName' and assign it a string value.
let studentName = "Alex";

// We declare a variable named 'age' and assign it a number value.
let age = 10;

// You can change the value of a 'let' variable later on.
age = 11;

// This variable stores a yes/no value (true or false).
let isStudent = true;

3. `let` vs. `const` vs. `var`: A Deeper Look at Variables

In JavaScript, there are three main ways to declare a variable. The one you choose is important because it changes how the variable behaves, especially with **scope** and **hoisting**.

Scope: Where a Variable Lives

Scope defines where a variable is accessible in your code. Think of it as a variable's "neighborhood."

  • **Global Scope:** Variables declared outside of any function or block live here. They can be accessed from anywhere in the code.
  • **Block Scope:** Variables declared with `let` and `const` are only available inside the block of code where they were created (a block is anything between curly braces `{}`, like an `if` statement or a `for` loop).
  • **Function Scope:** Variables declared with `var` are only available inside the function where they were created.

Hoisting: The "Lifting" of Declarations

Hoisting is JavaScript's way of moving variable and function declarations to the top of their scope before the code runs. This means you can sometimes use a variable before you declare it, but this can lead to unexpected results.

The `var` Keyword (The Old Way)

Variables declared with `var` are **function-scoped** and are fully **hoisted**. The variable's declaration is moved to the top, but its assignment stays put. This can be confusing.

// The code "as written"
console.log(carName); // This will output 'undefined'
var carName = "Tesla";

// How JavaScript "sees" it due to hoisting:
var carName; // The declaration is hoisted to the top.
console.log(carName); // carName exists, but it has no value yet, so it's 'undefined'.
carName = "Tesla"; // The value is assigned here.

The `let` Keyword (The Modern Way)

Variables declared with `let` are **block-scoped**. They are also hoisted, but they are not initialized. This creates a "temporal dead zone," which means you cannot access them before their declaration. This helps prevent bugs.

// The code "as written"
// This will cause an error!
// console.log(cityName); // This will cause an error: ReferenceError
let cityName = "New York";

// The difference: The variable exists in a "dead zone" until its declaration is reached.

The `const` Keyword (For Constants)

Variables declared with `const` are also **block-scoped**, but they have one key difference: their value **cannot be reassigned** after they are declared. You must assign a value when you declare them. `const` is great for values that should never change.

// A variable that will never change.
const birthYear = 1999;

// This would cause an error!
// birthYear = 2000; // This is not allowed because 'birthYear' is a constant.

Declaration vs. Assignment

It's important to understand the difference. You **declare** a variable when you introduce it for the first time (e.g., `let x;`). You **assign** a value when you give it a value (e.g., `x = 5;`).

let myNumber;      // Declaration
myNumber = 10;     // Assignment

let myNewNumber = 20; // Declaration AND Assignment in one line

4. Getting and Changing HTML Elements

This is how JavaScript connects to the HTML on the page. We use a special function to "get" an element and then we can change its properties.

Hello! My name is ...loading name....

The JavaScript code below will find the `` tag with the ID `name-display` and update its content.

// document: Refers to the entire HTML page.
// getElementById(): A function that finds an element by its ID.
let nameElement = document.getElementById('name-display');

// .textContent: A property that holds the text content of an element.
// We use the assignment operator (=) to change its text.
nameElement.textContent = "Alex";

5. Event Listeners: Responding to User Actions

An **event listener** is a piece of code that "listens" for a specific event to happen on the webpage, like a button being clicked or a key being pressed. When the event occurs, the listener runs a function that you define.

You have clicked the button 0 times.

The `addEventListener()` function is used to set this up. It takes two arguments:

  • The **event name** (e.g., `'click'`).
  • A **function** to run when the event happens. This is often called a "callback" function.
// First, get the button element by its ID.
const myButton = document.getElementById('my-button');

// Get the element to display the count.
const clickCountElement = document.getElementById('click-count');

// Create a variable to track the number of clicks.
let count = 0;

// Add a 'click' event listener to the button.
myButton.addEventListener('click', function() {
    // This code runs every time the button is clicked.
    count = count + 1; // Or simply: count++;
    clickCountElement.textContent = count;
    console.log(`Button clicked! The count is now ${count}.`);
});

6. Using the Console for Debugging

The `console.log()` function is a powerful tool. It prints a message to the browser's Developer Console, which is hidden from normal users. It's a great way for us to check our work and see what our variables hold.

To open the console, press `F12` on Windows or `Cmd + Option + J` on Mac.

// This will print the message "Hello from the console!" to the console.
console.log("Hello from the console!");

let score = 95;
// We can also print the value of a variable.
console.log("The current score is:", score);

7. JavaScript Arithmetic Operators

JavaScript uses the same mathematical symbols you already know to perform calculations.

  • `+` (Addition)
  • `-` (Subtraction)
  • `*` (Multiplication)
  • `/` (Division)
  • `%` (Modulus or Remainder)
// Addition:
let sum = 5 + 3; // sum is 8

// Subtraction:
let difference = 10 - 4; // difference is 6

// Multiplication:
let product = 6 * 5; // product is 30

// Division:
let quotient = 20 / 4; // quotient is 5

// Modulus:
let remainder = 10 % 3; // remainder is 1 (because 10 divided by 3 is 3 with a remainder of 1)
