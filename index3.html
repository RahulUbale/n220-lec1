JavaScript Execution Context

Explaining how JavaScript code is stored and executed

1. What is the Execution Context?

In JavaScript, all code runs inside a special environment called the Execution Context. You can think of it as a container where both data and instructions are handled.

The Execution Context has two main parts: Memory Component and Code Component.

2. Two Components of Execution Context

Memory Component

Stores variables and functions in the form of key-value pairs. It is also called the Variable Environment.

Code Component

Executes code line by line. It is also known as the Thread of Execution.

3. Synchronous & Single-Threaded Nature

JavaScript follows a synchronous single-threaded model. This means:

  • Only one command runs at a time.
  • Each line is executed in order.
  • The next line runs only after the current one finishes.

Example: If you have three console.log statements, they will run one after another, in sequence.

console.log("Step 1");
console.log("Step 2");
console.log("Step 3");

Quick Recap

  • Execution Context is the environment where JavaScript runs.
  • It has two parts:
    • Memory Component (Variable Environment)
    • Code Component (Thread of Execution)
  • JavaScript runs synchronously and is single-threaded.
